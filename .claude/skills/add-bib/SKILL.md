# add-bib

Add a single hand-supplied BibTeX entry to the thesis without ever duplicating a
reference that the pipeline already merged. The disambiguation is delegated to
`scripts/add_bib.py`, which reuses the exact tiered matcher from `merge_bibs.py`
(definitive shared identifier -> canonical field equality -> title-BoW Jaccard
candidate). See the *Bibliography* notes in `CLAUDE.md`.

`papers-bibliography.bib` is pipeline-generated and must never be hand-edited;
new manual entries land in `extra-bibliography.bib`, a second `\addbibresource`
that this skill creates and wires in on first use.

## Input

A single BibTeX entry, e.g.:

```bibtex
@book{pierce2002types,
  author    = {Benjamin C. Pierce},
  title     = {Types and Programming Languages},
  year      = {2002},
  publisher = {MIT Press},
}
```

If the user gives several entries at once, run the procedure once per entry.

## Procedure

1. **Write the entry to a temp file** with the Write tool, e.g. `/tmp/add-bib-input.bib`.
   Use the entry verbatim — do not reformat or invent fields.

2. **Run the query script** from the repo root:

   ```
   uv run scripts/add_bib.py /tmp/add-bib-input.bib
   ```

   It prints one JSON object and exits 0. Parse it.

3. **Branch on `status`:**

   - **`found`** — the entry already exists. Report and stop:

     > Yes, it is there under this code: `<key>` (in `<source>`).

   - **`candidate`** — a same-title entry exists but with no shared DOI/eprint,
     so it's ambiguous. Show the user both `existing_preview` and `new_preview`
     and ask whether they are the same work (Jaccard `<jaccard>`). Use
     AskUserQuestion.
     - **Same work** → report as found:
       > Yes, it is there under this code: `<key>` (in `<source>`).
       Stop.
     - **Different works** → continue to step 4 (treat as not found).

   - **`not_found`** — continue to step 4.

   - **`error`** — report `message` and stop (commonly: input had 0 or >1 entries).

4. **Append to `extra-bibliography.bib`** (repo root):
   - If it does **not** exist, create it with the header comment below followed by
     the entry.
   - If it **does** exist, append the entry preceded by one blank line (Edit/Write).

   Header for a freshly created file:

   ```
   % Hand-added references not covered by the arXiv pipeline (papers-bibliography.bib).
   % Loaded as a second \addbibresource in thesis.tex. Safe to hand-edit.
   ```

5. **Wire into `thesis.tex` (idempotent):**
   - If `thesis.tex` already contains `\addbibresource{extra-bibliography.bib}`,
     do nothing here.
   - Otherwise insert that line immediately **after**
     `\addbibresource{papers-bibliography.bib}`.

6. **Report** concisely what changed:

   > Added `<key>` to `extra-bibliography.bib`.
   > (plus, if applicable) Wired `extra-bibliography.bib` into `thesis.tex`.

## Constraints

- Never modify `papers-bibliography.bib` — it is regenerated by `merge-bibs` and
  hand-edits get wiped. See [[feedback-dont-rerun-editorial-scripts]].
- Never touch anything under `arxiv-papers/`. See [[feedback-arxiv-papers-readonly]].
- `scripts/add_bib.py` is query-only; all file writes are done here via Write/Edit
  so they stay visible and reversible.
- Don't reformat the user's BibTeX or fabricate fields; preserve the entry as given.
- Trust the script's matcher — don't second-guess a `found`/`not_found` verdict;
  only the `candidate` tier needs a human call.
